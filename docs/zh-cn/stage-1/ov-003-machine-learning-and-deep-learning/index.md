## 本节导入

相册应用自动把家人的照片归到一组，输入法预测你要打的下一个词，地图软件预估到达时间——这些功能背后没有一条条人工写死的判断规则。支撑它们的是同一件事：**让程序从数据中自己找出规律**，这就是机器学习。

传统软件开发把人的经验翻译成明确规则交给机器执行。可是"垃圾邮件千变万化""人脸的角度光线各不相同"这类问题，规则永远写不完、也改不过来。机器学习换了一条路：给机器足够多的例子，让它自己总结出判断依据。

本节沿两条主线展开。前半段回答"机器怎样从数据中学习、我们怎样判断它学得好坏"，核心是监督学习；后半段看深度学习如何把"自动学习"推进到特征层面，让机器自己发现"什么值得看"。这两块内容是后续理解大模型、智能体的共同地基。

## 核心内容

### 机器学习：从数据中学习规律

传统程序的工作方式是"人定规则、机器执行"。机器学习颠倒了这个次序：先给机器大量"输入—答案"的样例，由算法从中归纳出一个映射函数 f，再用这个函数去处理没见过的新输入。识别照片里是人还是狗、判断一段评论是好评还是差评，本质上都是在学这样一个 f。

按照"给机器什么样的数据"来分，机器学习有几种基本范式：

- **监督学习**：数据带标签，例如"这封邮件是垃圾邮件"。典型任务是分类和回归。
- **无监督学习**：数据没有标签，让算法自己发现结构，典型任务是聚类和降维。
- **强化学习**：没有现成答案，智能体在与环境的交互中根据奖励信号学会决策，下棋和机器人控制是典型场景。
- **半监督学习**：少量带标签数据加大量无标签数据混合使用，贴近标注成本高的现实。

本节聚焦监督学习：它是理解深度学习与大模型的基础，工业界大多数落地系统也从它做起。

### 监督学习的三个要素

要让"从例子中学习"真正跑起来，三样东西缺一不可：

1. **标注数据**：成对的"输入—标签"样本，例如一万张标好"猫/狗"的图片。数据的数量和质量直接决定上限。
2. **学习模型**：一族带参数的候选函数，规定了 f 可能长什么样——可以是一条直线，也可以是一个深层网络。
3. **损失函数**：给"预测得有多差"打分的尺子。预测值和真实标签差得越远，损失越大。

有了这三样，训练过程可以概括成一句话：**调整模型参数，让全部训练样本上的平均损失尽可能小**。这个平均损失有个正式名字，叫经验风险，因此这种训练方式称为经验风险最小化。

### 训练、验证与测试：学得好还要用得好

一个关键追问随之而来：模型在见过的数据上表现好，能保证它遇到新数据还行吗？规范做法通常把数据分成三份：**训练集**用来学习参数，**验证集**用来选择模型、阈值和超参数，**测试集**留到方案冻结后做最终评估。若反复根据测试集改模型，测试集也会被“间接学会”，成绩就不再代表未见数据上的泛化能力。

这对区分引出两个概念：**经验风险**是模型在训练数据上的平均损失；**期望风险**是模型面对真实数据分布时的预期损失。样本足够多时两者会趋于一致，但现实中训练样本总是有限的，一味压低经验风险就会出现两种典型病症：

- **过拟合（过学习）**：训练集上近乎完美，测试集上一塌糊涂。模型把训练样本的噪声和巧合都背了下来，像只会背答案的学生，换一套题就露馅。
- **欠拟合（欠学习）**：训练集上都学不好，模型太简单或训练不充分，像还没入门的学生。

治过拟合的经典药方是**结构风险最小化**：在经验风险后面加上一项模型复杂度惩罚（正则化项），在"拟合数据"和"保持简单"之间找平衡。这与奥卡姆剃刀的直觉一致——能用简单模型解释的，就别用复杂模型。今天训练深度网络时用的各种正则化手段，都是这一思想的延续。

### 线性回归：最经典的入门模型

最能把上面这套流程讲透的模型是**线性回归**。19 世纪，英国学者高尔顿研究父母身高与子女身高的关系时发现：高个子父母的子女平均而言也高，但会向人群平均值靠拢——"回归"一词就源于这个现象。他得到的关系可以写成一条直线：子女身高 = 33.73 + 0.516 × 父母平均身高（单位为英寸）。

换一个更现代的例子：用气温预测森林火灾的过火面积。把历史记录画成散点图，找一条最能贴合这些点的直线 y = ax + b。"学习"在这里就是从数据中确定参数 a 和 b——标准做法是让所有样本的预测误差平方和最小，这就是**最小二乘法**。当影响因素多于一个（再加上风力、湿度），模型扩展为多元线性回归，思路完全不变。

线性回归虽然简单，却完整走通了监督学习的通用套路：**定模型形式 → 定损失函数 → 用数据求参数**。后面所有更复杂的模型，包括深度网络，走的都是同一条流水线，只是把"模型形式"换得更强大。

### 从回归到分类：逻辑斯蒂回归与决策树

回归输出连续数值，分类输出离散类别，监督学习的任务基本落在这两类里。想用普通线性回归直接做二分类会遇到一个根本问题：预测值没有被限制在 0 到 1 之间，也不对应适合二分类的似然模型。离群点同样可能影响参数，但它不是逻辑斯蒂回归自动解决的问题。

**逻辑斯蒂回归**在线性得分外面使用 Sigmoid 函数，把任意实数映射到 0 和 1 之间，并通过对数似然或交叉熵来学习参数。在模型假设与数据条件合适时，输出可解释为正类的条件概率；实际系统若需要可靠概率，还应检查校准情况。它至今仍是工业界二分类问题的常用基线，但并不天然具备离群点鲁棒性。

**决策树**代表另一条思路：像人做判断一样，一步一步问问题。看一个直观例子——是否见一位相亲对象：先看年龄段，再看是否有房产、是否结过婚，每个属性问一次，顺着树枝走到叶子就得到结论。

先问哪个属性？这里用到**信息熵**——不确定性的度量。熵越大，样本越混杂；一次好的提问应该让划分后的子集尽可能"纯"。每次选择能带来最大**信息增益**的属性作为当前节点，自顶向下把树长出来。

决策树最大的优点是**可解释**：每一步判断都能讲清楚理由。信贷风控、医疗辅助判断这些需要"说得清依据"的场景，决策树及其衍生算法长期占有一席之地。

### 经典模型速览：LDA、AdaBoost 与支持向量机

监督学习几十年的发展攒下了一批各有绝活的经典模型，了解它们的核心思想，就能看懂大部分传统机器学习系统的设计逻辑。

**线性判别分析（LDA）** 做的是带标签的降维：把高维数据投影到低维空间，让同类样本聚得更拢、异类样本离得更远——"类内方差小、类间间隔大"。

**AdaBoost** 走"集成"路线：单个弱分类器只比瞎猜强一点，把一系列弱分类器组合起来，就能得到一个强分类器——三个臭皮匠顶个诸葛亮。它的两个关键动作：每一轮提高上一轮被分错样本的权重，让后续弱分类器专攻难题；最后按各自的错误率加权投票。

**支持向量机（SVM）** 直面"哪条分界线最好"的问题：能把两类分开的超平面有无数个，SVM 挑使两类样本离边界最远的那一个——间隔最大，对新样本最稳。配合核函数，它还能把线性不可分的数据映射到高维空间再切分。SVM 也是结构风险最小化思想最彻底的践行者。

顺带认识一对概念：以上模型大多属于**判别模型**，直接学"输入到类别"的边界；**生成模型**（如朴素贝叶斯、隐马尔可夫模型）则学习数据本身的分布规律，还能反过来合成新数据——这条线索到深度生成学习一节还会回来。

这批经典模型有一个共同的天花板：**特征仍然主要靠人来设计**。图像里该提取什么信息、文本该统计哪些量，都依赖专家经验。深度学习登场，针对的正是这块短板。

### 深度学习：让机器自己学特征

在深度学习之前，一个图像识别系统是"分段"组装的：人工设计特征提取（边缘、纹理、视觉词典），再接一个分类器，每一段单独调优。深度学习改成**端到端**：原始像素直接进网络，识别结果直接出来，中间的特征全部由网络自己学。

网络内部发生的事情可以理解为**逐层抽象**：低层单元常响应边缘和亮暗，中层组合出角点和部件，高层形成与对象类别相关的表示。这种层级处理受到生物视觉研究启发，也存在有限类比，但人工网络并不是视觉皮层的等价模型。

网络的基本单元常被称为**人工神经元**。1943 年 McCulloch 和 Pitts 受生物神经元"兴奋/抑制"机制启发提出 MCP 模型：对输入加权求和，再经过一个激活函数决定输出。今天许多网络基本单元仍可抽象为线性组合与非线性变换，但注意力、归一化、门控和其他结构已经远超最初的 MCP 模型。

**激活函数**负责引入非线性——没有它，再多层网络叠起来仍等价于一个线性模型。Sigmoid、tanh、ReLU 是常用选择；多分类输出层常用 softmax 把各类得分变成一组概率。

只有输入层和输出层的**单层感知机**能模拟逻辑与、或、与非，却对"异或"无能为力，因为异或线性不可分——这个局限曾让神经网络研究陷入低谷。破局的办法是加**隐藏层**：多层感知机让上一层的输出作为下一层的输入，层与层全连接、信息单向流动，这类结构统称**前馈神经网络**。层数够深、神经元够多，它就能逼近极其复杂的函数。

### 神经网络怎么训练：梯度下降与误差反向传播

网络结构定了，剩下的问题是：这么多参数（权重）怎么定？答案仍是监督学习那套流程——定义损失函数，然后优化它。回归任务常用均方误差；分类任务常用交叉熵，它度量预测概率分布与真实分布的差距。

优化的主力算法是**梯度下降**。把损失函数想象成参数空间里的一片山地，当前参数是站立的位置：梯度指向上坡最陡的方向，往梯度的反方向挪一小步，损失就下降最快。步子的大小叫学习率，一步步走下去，直到走进谷底附近。

深层网络的参数分布在一层层结构里，输出端算出的误差如何反馈给前面每一层？**误差反向传播**（BP）解决了这件事：利用链式求导法则，把输出误差从后向前逐层传递，算出每个参数对总误差该负多大"责任"，再按责任大小更新。1986 年反向传播的推广让训练多层网络真正变得可行，也为后来的深度学习铺平了路。

把这一节收拢成一句话：**机器学习的能力在于拟合和优化**——网络结构负责提供足够强的表达能力，梯度下降加反向传播负责把参数调到位。

### 卷积神经网络：看懂图像

把一张图像直接摊平接进全连接网络，参数会瞬间爆炸：一张 1000×1000 的图片仅一层就要处理百万维输入。图像有两个可以利用的特点——**模式是局部的**（一条边缘只占几个像素），**模式会在不同位置重复出现**。**卷积神经网络**（CNN）把这两点变成了结构设计。

核心是**卷积操作**：用一个小小的卷积核（如 3×3 的权重矩阵）在图像上滑动，每停一处就与覆盖区域做加权求和，输出一张特征图。同一个卷积核的参数在整张图上**共享**，参数量骤减，而且它学到的模式在图像任何位置都能被检测到。

不同的卷积核提取不同特征：有的对竖直边缘敏感，有的响应特定纹理——这与视觉神经细胞的"特征选择性"相互呼应。多个卷积核并行工作，就得到多张互补的特征图。

卷积之后常接**池化操作**：用一个区域的统计值（最大值或平均值）代替整个区域，把特征图压小。图像冗余大，池化在大幅降低计算量的同时保留了主要信息，还让特征对小幅平移不那么敏感。

把"卷积 → 激活 → 池化"反复堆叠，最后接全连接层输出类别，就是完整的 CNN 图像分类流水线。前面几层学到边缘纹理，越往后特征越抽象，与"逐层抽象"的图景严丝合缝——这正是"深度"二字的含义。

训练深层网络同样要防过拟合，常用手段包括 Dropout（训练时随机丢弃部分神经元）、批归一化和 L1/L2 正则——结构风险最小化的思想在深度时代照常生效。

### 序列与生成：循环网络、生成对抗与词向量

图像一次性输入就好，语言、语音、时序数据却有前后依赖。**循环神经网络**（RNN）为此在网络里加了一条"记忆"通路：每读一个词，把当前输入和上一步的隐状态一起加工，隐状态就像一路携带的笔记，让后面的判断能利用前面的信息。

序列一长，RNN 的梯度在层层回传中容易消失，早先的信息就被"忘掉"。**长短时记忆网络**（LSTM）用输入门、遗忘门、输出门管理一个内部记忆单元，学会该记什么、该忘什么，明显改善了长序列建模，一度是机器翻译和语音识别的主力结构。

前面讲的模型都在做判断，**深度生成学习**反过来让网络学会"创造"。**生成对抗网络**（GAN）的设计精彩之处在于左右互搏：生成器负责伪造数据，判别器负责鉴别真伪，两者在博弈中共同进步，直到生成的样本以假乱真。

在语言一侧还有一块重要拼图：**词向量**。Word2Vec 把每个词映射为一个稠密向量，语义相近的词在向量空间中彼此靠近，"用数值表示语义"从此成为可能。

到这里可以把整节课串起来了：监督学习给出"数据 + 损失函数 + 优化"的方法论，深度学习提供了能自动学特征的强大模型形式。今天的大语言模型仍以这套框架为基础——大规模数据提供训练样本，预测下一个 token 构成重要训练目标，梯度方法驱动优化，Transformer 负责处理上下文关系。下一节将进一步拆解这些概念。

### 关键概念

- **监督学习**：用带标签数据学习"输入到输出"映射的学习范式。
- **训练、验证与测试数据**：训练集学参数，验证集做选择，测试集在方案冻结后检验泛化。
- **经验风险**：模型在训练数据上的平均损失。
- **结构风险最小化**：经验风险加复杂度惩罚，在拟合与简单之间求平衡。
- **线性回归**：用线性方程刻画变量关系的最基础回归模型。
- **前馈神经网络**：层间全连接、信息单向流动的多层网络结构。
- **梯度下降**：沿损失下降最快的方向迭代更新参数的优化算法。
- **误差反向传播**：用链式法则把输出误差逐层回传、分摊到每个参数的训练方法。
- **卷积神经网络**：以卷积和池化为核心、擅长图像等网格数据的深度网络。

<h3>案例 1：垃圾邮件过滤</h3>
<p><strong>场景：</strong>邮件系统需要自动判断新到的一封邮件是否为垃圾邮件。</p>
<p><strong>问题：</strong>垃圾邮件写法不断翻新，人工规则列表永远追不上变化。</p>
<p><strong>AI 做法：</strong>用大量已标注邮件训练分类模型（逻辑斯蒂回归即可作基线），并留出测试数据检验对新邮件的识别率。</p>
<p><strong>结果：</strong>模型输出"是垃圾邮件的概率"，按阈值自动过滤，准确率随数据积累持续提升。</p>
<p><strong>人工复核点：</strong>误拦正常邮件代价高，需要保留申诉通道，并定期用新样本重新训练。</p>
</article>
<article class="ai-course-case">
<h3>案例 2：森林火灾面积预测</h3>
<p><strong>场景：</strong>林业部门想根据气象条件预估火灾可能波及的面积，提前调配力量。</p>
<p><strong>问题：</strong>影响因素多、关系不直观，凭经验估算误差大。</p>
<p><strong>AI 做法：</strong>用历史记录建立回归模型：先从"气温 → 过火面积"的一元线性回归入手，再加入风力等因素扩展为多元回归。</p>
<p><strong>结果：</strong>给出任意气象条件即可得到过火面积的量化预估，为资源调度提供依据。</p>
<p><strong>人工复核点：</strong>极端天气超出历史数据分布时模型不可靠，需人工研判并把新数据补入训练集。</p>
</article>

<h3>案例 3：手写数字识别</h3>
<p><strong>场景：</strong>银行票据和邮政编码上的手写数字需要机器自动读取。</p>
<p><strong>问题：</strong>每个人笔迹不同，人工设计的特征始终覆盖不了书写变化。</p>
<p><strong>AI 做法：</strong>用卷积神经网络端到端训练：卷积层自动学习笔画特征，池化层压缩冗余，全连接层输出 0 到 9 的概率。</p>
<p><strong>结果：</strong>经典的 LeNet 网络早在上世纪九十年代就投入美国支票识别系统，是深度学习最早的规模化应用之一。</p>
<p><strong>人工复核点：</strong>低置信度样本转人工录入，识别错误会直接造成金额差错，阈值设置要保守。</p>
</article>

## 动手实践

- 用表格软件对 10 组"气温—用电量"数据画散点图并添加线性趋势线，读出斜率和截距，说明它们各自的现实含义。
- 把你工作中的一个真实问题改写成监督学习任务：写清输入特征是什么、标签是什么、训练数据与测试数据分别从哪里来。
- 打开浏览器版 [Teachable Machine](https://teachablemachine.withgoogle.com/)，训练一个两类图像分类器：先每类只给 5 张样本，再增加到 30 张，对比测试效果，体会数据量与过拟合的关系。
- 挑一个你熟悉的自动化判断场景（如信贷审批、内容审核），分析它更适合决策树还是神经网络，从"可解释性"和"数据规模"两个角度写出理由。

完成后保留你的数据、步骤和结论，下一节讨论 AI 能力边界时会用它们做对照材料。

## 课后复盘

- 训练数据与测试数据为什么必须分开？只看训练集上的成绩会掉进哪个陷阱？
- 经验风险最小化为什么不够，结构风险最小化补上了什么？
- 线性回归、逻辑斯蒂回归、决策树各适合什么样的问题？
- 单层感知机的局限是什么，多层前馈神经网络靠什么突破了它？
- 梯度下降和误差反向传播在训练中各自承担什么角色？
- 卷积神经网络的"参数共享"解决了全连接网络的什么毛病？

把每个答案压缩成一两句自己的话，再回到三个案例里检验：你能不能指出每个案例分别用到了本节的哪些概念。
