# 本地课程维护方案

## 核心契约

课程共 4 章、27 节。每节都有两份发布内容：

1. `docs/zh-cn/stage-N/<lesson>/index.md`
2. `docs/public/course-slides/stage-N/<lesson>/slides.pptx`

`course-catalog.json` 同时记录网页链接、发布 PPTX、本地原稿以及原稿选页范围。这是课程映射的唯一真实来源。

## 更新流程

### 新增或重新拆分课件

1. 确定课程 ID、章节、slug 和 Markdown 路径。
2. 在课程清单中记录 `source_pptx` 与 `source_slides`。
3. 运行 `pnpm courseware:build -- --lesson <ID>`，从原稿无损复制选定页面。
4. 检查生成的逐页预览，排除联系方式、答辩结束页、过期或偏题内容。
5. 运行 `pnpm generate` 和 `pnpm verify`。

### 修改讲义或课件

- 讲义变更：直接修改对应 Markdown，保留 `authored: true` 和课程 ID。
- 课件变更：先修改本地原稿或调整 `source_slides`，再重新拆分本节 PPTX。
- 结构变更：同步更新课程清单、Markdown、PPTX 和导航，不保留旧链接兼容层。

## 发布边界

- 网页“幻灯片”只链接本地 PPTX，不内嵌播放器。
- 发布课件必须保留原生 PowerPoint 结构，不使用整页图片替代。
- 每份 PPTX 必须小于 95 MiB，以留出 GitHub 单文件限额余量。
- 原稿目录留在本机且由 `.gitignore` 排除；GitHub Pages 只发布节级 PPTX。
