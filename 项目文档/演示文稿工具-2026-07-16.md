## 实际能力

BananaPPT 是一个纯前端 HTML 单页工具。它把用户文本、预设风格提示词和参考图提交给第三方图像生成 API，生成结果是可下载的单页 PNG 图片。

它不是完整演示文稿生成器：

- 不生成可编辑的 PPTX。
- 不生成多页幻灯片结构、讲者备注或母版。
- 文本、图表、箭头和卡片都固化在图像中，无法在 PowerPoint 里逐项修改。

因此，X 帖文的“一键生成 PPT”更准确的表述应是“一键生成 PPT 风格的单页静态图”。

## 与 ai-course 的适配度

| 用途 | 适配度 | 说明 |
| --- | --- | --- |
| 风格提示词参考 | 中 | 仓库公开了多种商务、学术、政务和汇报风格的提示词，可用来研究视觉语言 |
| 快速视觉草图 | 中 | 可用于尝试单页信息密度和构图，成果需人工重建 |
| 课程独立插图 | 低到中 | 只有在文字、数据、版权和可读性都通过复核后才能作为独立图解 |
| 生成 27 份课程 PPTX | 不适合 | 输出是整页栅格图，违反本项目“保留可编辑 PPTX 结构、不将整页幻灯片栅格化发布”的契约 |

## 安全与授权边界

- X 原帖公开了一段共享体验 Key。本整理不保留、不调用、不验证该 Key。
- 页面会把用户输入的 Key 存入浏览器 `localStorage`，并发送到代码中硬编码的第三方 API 域名。不应在其中输入正式密钥、客户材料、未公开数据、个人信息或内部课件。
- 截至核验日，GitHub 仓库未声明软件许可证。“代码可见”不等于获得了复制、修改、再发布或商业使用授权。
- 仓库没有发布正式 Release，也没有后端服务的完整源码、隐私说明或服务承诺，不应视为稳定的生产工具。

## 结论

将 BananaPPT 保留为“风格提示词与单页视觉草图参考”，不纳入 ai-course 的正式 PPTX 生成、课件拆分或发布链路。
