# 课程研究与章节结构

## 调研范围

本轮结构复核以 2021—2026 年公开课程、原始论文和权威治理框架为主。结构参考包括 Stanford CME 295 与 CS25 的 Transformer/LLM 路径、Berkeley 2024 LLM Agents 课程、Stanford CS223A 机器人学课程，以及 NIST AI RMF/GenAI Profile。

## 第一章：人工智能概述

学习路径：“人工智能系统与社会问题 → 机器学习与深度学习 → Transformer 与大模型 → 开源模型生态 → 提示与多模态工作流 → 安全与责任”。

1. OV-001：人工智能引论，建立感知—推理—行动框架。
2. OV-002：发展史与社会影响，用任务变化代替简单“岗位消失”叙事。
3. OV-003：数据、特征、模型、训练/测试、过拟合与深度表征。
4. OV-004：token、注意力、Transformer、预训练、后训练和能力边界。
5. OV-005：DeepSeek 与开源模型，比较 API、开源权重与本地部署。
6. OV-006：提示词、AIGC 与多模态内容工作流。
7. OV-007：幻觉、提示注入、权限边界、证据与人工责任。

主要来源：

- Stanford CME 295 Syllabus: https://cme295.stanford.edu/syllabus/
- Stanford CS25: https://web.stanford.edu/class/cs25/
- InstructGPT: https://arxiv.org/abs/2203.02155
- DeepSeek-V3: https://arxiv.org/abs/2412.19437
- RAG Survey: https://arxiv.org/abs/2312.10997
- Hallucination Survey: https://arxiv.org/abs/2311.05232
- NIST GenAI Profile: https://www.nist.gov/publications/artificial-intelligence-risk-management-framework-generative-artificial-intelligence

## 第二章：AI 智能体

学习路径：“单智能体闭环 → 工具、记忆与规划 → 多智能体协作 → 编程智能体 → 轨迹、评测与人工复核”。这与 Berkeley LLM Agents 2024 从推理、工具和基础设施到评测、安全和人机交互的结构一致。

1. AG-001：Observe—Reason—Act—Evaluate 循环与 ReAct。
2. AG-002：工具契约、MCP、短期/长期记忆、计划与失败分支。
3. AG-003：角色拆分、交接协议、冲突解决和共享进度。
4. AG-004：需求、代码生成、测试、审查与回滚。
5. AG-005：trace、结果/过程评测、风险阈值与人工检查点。

主要来源：

- Berkeley LLM Agents Fall 2024: https://rdi.berkeley.edu/llm-agents/f24
- ReAct: https://arxiv.org/abs/2210.03629
- Toolformer: https://arxiv.org/abs/2302.04761

## 第三章：具身智能

学习路径：“身体—环境—任务闭环 → 机器人平台 → 多模态感知 → 运动学/动力学与学习控制 → ROS2 实训 → 开放式和群体具身系统”。它将 Stanford 机器人课程的运动学、雅可比、动力学和控制主线，与 RT-2/OpenVLA 的视觉—语言—动作路线连接。

1. EM-001：感知—推理—执行—反馈的具身闭环。
2. EM-002：人形/四足平台的任务、性能、成本与运维权衡。
3. EM-003：视觉、深度、激光雷达、触觉与主动感知。
4. EM-004：步态控制、强化学习、模仿学习、仿真与 sim-to-real。
5. EM-005：G1/Go2、ROS2 话题、传感器链路、安全状态机与技能包。
6. EM-006：开放动态环境、多模态推理、自我进化与群体协同。

主要来源：

- Stanford Robotics teaching: https://ai.stanford.edu/manips/teaching.html
- Stanford CS223A: https://cs.stanford.edu/group/manips/teaching/cs223a/
- RT-2: https://arxiv.org/abs/2307.15818
- OpenVLA: https://arxiv.org/abs/2406.09246

## 第四章：行业应用

本章不再以“行业案例堆叠”组织，而是为 9 节统一使用“业务基线 → 数据与权限 → 模型/工具链 → 人工责任 → 指标与回滚”框架，对齐 NIST AI RMF 的 Govern—Map—Measure—Manage 方法。

1. IN-001：行业 AI 价值链、ROI 和试点门槛。
2. IN-002：高校教学与科研的检索、引用、复现与诚信边界。
3. IN-003：政务与乡村场景的授权、证据、行政责任和普惠性。
4. IN-004：企业办公、RAG、知识有效期和审批链。
5. IN-005：制造、能源和供应链的传感—预测—决策—执行闭环。
6. IN-006：交通与智慧机场的定位、规划、避障与安全管理。
7. IN-007：应急与火灾调查的证据链、假设竞争和人工复核。
8. IN-008：海洋与时空智能的多源数据、坐标尺度和不确定性。
9. IN-009：文旅、会展和智慧建筑的人—空间—设施协同。

主要来源：

- NIST AI RMF Core: https://airc.nist.gov/airmf-resources/airmf/5-sec-core/
- NIST GenAI Profile: https://www.nist.gov/publications/artificial-intelligence-risk-management-framework-generative-artificial-intelligence
- UNESCO GenAI in Education and Research: https://www.unesco.org/en/articles/guidance-generative-ai-education-and-research?hub=67098
