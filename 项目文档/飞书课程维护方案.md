# 飞书课程维护方案

## 目标结构

课程主线保持四章、二十七节。每节课程具有两个同步维护对象：

1. 网站中的 Markdown 讲义；
2. 飞书知识库中的原始 PPTX 或原生 Slides。

课程映射以 `课程PPT框架/source/course-catalog.json` 为准。`slides.url` 与 `slides.wiki_url` 指向同一份原始飞书课件，`slides.local_pptx` 指向用于生成和校准 Markdown 的本地同步文件。

## 同步流程

新增课程时，先确定课程 ID 和章节，再把已有原始课件整理到对应飞书章节，最后记录飞书链接和本地 PPTX 路径并运行 `pnpm generate`。不要在仓库中重新生成 PPT。

修改课程时，先在飞书精修原始课件，再从同步后的本地 PPTX 校准 Markdown。修改后运行 `pnpm courseware:verify:remote` 检查远端映射，再运行 `pnpm verify` 检查网站、媒体与课程质量。

删除课程时，同步处理课程清单、Markdown、网站导航和飞书原始课件。不要保留失效播放器字段、旧模块 PPTX 链接或兼容分支。

## 播放与权限

课堂播放直接打开飞书原始课件。网站不内嵌播放器，也不托管 PPTX。

课程内容整理、隐私检查、第三方素材授权和页面验收完成后，才单独调整公开权限。公开时只开放“持链接可阅读”，编辑、转发、安全设置和外部邀请仍保持受限。

## 媒体边界

网站保留经审查的独立图解和视频，它们用于 Markdown 讲解，不是幻灯片整页截图。禁止将 PPT 或 PDF 整页栅格化后作为网页课程内容发布。
